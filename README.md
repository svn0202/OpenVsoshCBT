# OpenVsoshCBT

OpenVsoshCBT — открытая система компьютерного тестирования для проведения
олимпиад, экзаменов, тренировочных работ и других видов контроля знаний в
локальной сети или через интернет.

Проект является форком [TCExam](https://github.com/tecnickcom/tcexam) и на
данном этапе основан на TCExam 17.2.4. Он адаптируется для русскоязычного
интерфейса, PostgreSQL и развёртывания на РЕД ОС — RHEL/CentOS-совместимой
серверной среде.

В качестве источника проверенных пользовательских сценариев используется
свободная ветка TMF/TMFCBT Мамана Сулаемана. Полезные возможности TMFCBT
переносятся по одной на актуальное ядро TCExam с ревью и тестами; старая
кодовая база TCExam 14.3.0 в этот репозиторий целиком не сливается.

## Текущее состояние

- современное ядро TCExam 17, PHP 8.2+ и зависимости Composer;
- PostgreSQL, MySQL/MariaDB и Oracle, интеграционные тесты БД;
- безопасная конфигурация экземпляра вне Git;
- HTML5, улучшенная доступность и актуальный PDF-движок;
- Docker-окружение и неинтерактивная установка;
- сохранённая совместимость с официальным upstream;
- начата документация для РЕД ОС и олимпиадного сценария;
- перенос функций TMFCBT ещё не завершён: текущий релиз нельзя считать
  функциональной заменой рабочей TMFCBT-установки.

## Быстрый запуск для оценки

Требуется Docker с Compose:

```sh
make up
```

Приложение будет доступно на <http://localhost:8080/>. Стандартная тестовая
учётная запись TCExam — `admin` / `1234`; измените или удалите её сразу после
первого входа. Для реального развёртывания используйте HTTPS и отдельную БД.

Ручная установка:

```sh
composer install
make lang
```

После этого следуйте [официальному руководству установки](install/README.md)
и [инструкции OpenVsoshCBT для РЕД ОС](docs/admin/deployment-redos.md).

## Конфигурация и секреты

В Git находятся только шаблоны `admin/config.default`,
`public/config.default` и `shared/config.default`. Установщик создаёт рабочие
`*/config` отдельно; эти каталоги игнорируются Git. Там остаются пароль БД,
SMTP-реквизиты, `K_RANDOM_SECURITY`, локальные пути и другие настройки
конкретного сервера.

Не коммитьте рабочие конфиги, `.env`, базы, ответы участников, выгрузки,
резервные копии и приватные ключи. Демонстрационный сертификат TCExam оставлен
для тестов намеренно и не подходит для доверенной подписи.

## Направление развития

В OpenVsoshCBT будут добавляться необходимые нам изменения для российских
олимпиад. Публично описанные функции коммерческих или закрытых редакций можно
использовать как идеи и требования для независимой реализации. Закрытый код,
бинарные пакеты и материалы с неясными правами не переносятся.

Каждая функция проходит путь: сценарий → сравнение с upstream → тест на
PostgreSQL → реализация → нагрузочная и аварийная приёмка.

## Связь с upstream

```text
origin    git@github.com:svn0202/OpenVsoshCBT.git
upstream  https://github.com/tecnickcom/tcexam.git
```

Обновления TCExam сначала подтягиваются в отдельную ветку, затем проходят
тесты OpenVsoshCBT. Функциональные изменения не должны смешиваться с
синхронизацией upstream в одном коммите.
