# Расчёт баллов

Эта страница описывает алгоритм TCExam 17.2.4 и сверена с
`F_updateQuestionLog()` в `shared/code/tce_functions_test.php`.

## Общие параметры

Для теста задаются:

- `R` — базовый балл за правильный ответ (`test_score_right`);
- `W` — балл за неправильный ответ (`test_score_wrong`);
- `U` — балл за отсутствие ответа (`test_score_unanswered`);
- `D` — сложность конкретного вопроса (`question_difficulty`).

Максимальный, штрафной и «без ответа» баллы вопроса:

```text
QR = R × D
QW = W × D
QU = U × D
```

Значения `W` и `U` могут быть нулевыми или отрицательными. Итоговый балл теста
— сумма сохранённых баллов вопросов. Проходной порог сравнивается с этой
суммой.

## MCSA — один вариант

- выбран правильный вариант: `QR`;
- выбран неправильный вариант: `QW`;
- вариант не выбран: `QU`.

Пример: `R = 2`, `W = -0,5`, `U = 0`, `D = 3`. Тогда вопрос приносит `6`
баллов за правильный выбор, `-1,5` за неправильный и `0` за пропуск.

## MCMA — несколько вариантов

Пусть у вопроса `N` вариантов. Каждый вариант оценивается как отдельное
решение «выбран/не выбран»:

- правильный вариант выбран — верное действие;
- правильный вариант не выбран — ошибочное действие;
- неправильный вариант выбран — ошибочное действие;
- неправильный вариант не выбран — верное действие;
- явно оставленный без ответа вариант — действие «без ответа».

Для верного действия берётся `QR`, ошибочного — `QW`, отсутствующего — `QU`.
После обработки всех вариантов поведение зависит от настройки частичного
балла.

### Частичный балл включён

```text
score = round((сумма баллов N действий) / N, 3)
```

Пример: четыре варианта, `QR = 4`, `QW = 0`. Участник правильно обработал три
варианта и ошибся в одном:

```text
(4 + 4 + 4 + 0) / 4 = 3 балла
```

Важно: невыбранный неправильный вариант тоже является правильным действием.
Поэтому при отображении MCMA обычными checkbox отсутствие отметки не всегда
означает отсутствие ответа на весь вопрос.

### Частичный балл выключен

- если все `N` действий правильны — `QR`;
- если все действия имеют состояние «без ответа» — `QU`;
- во всех остальных случаях — `QW`.

Это режим «всё или ничего».

## ORDER — упорядочивание

Каждый элемент сравнивается с правильной позицией:

- позиция совпала — `QR`;
- позиция не совпала — `QW`;
- позиция не задана — `QU`.

Далее сумма делится на число элементов при включённом частичном балле либо
применяется правило «всё или ничего», как для MCMA.

## TEXT — свободный/краткий ответ

Если введён непустой текст, TCExam сначала устанавливает балл в состояние,
требующее оценки. Затем он сравнивает ответ с включёнными правильными
вариантами короткого ответа:

- пробелы по краям удаляются;
- при `K_SHORT_ANSWERS_BINARY = true` сравнение чувствительно к регистру;
- при `false` регистр игнорируется;
- при точном совпадении назначается `QR`;
- без совпадения ответ остаётся для ручной проверки.

Эксперт может назначить `QR`, `QW`, `QU` или произвольный балл и комментарий.

## Максимальный балл теста

При настройке набора вопросов TCExam рассчитывает максимум как сумму:

```text
R × сложность × количество вопросов
```

по всем наборам. После изменения `R`, сложности или состава банка проверьте
максимум и проходной порог повторно.

## Практические проверки

Перед олимпиадой создайте контрольную таблицу ожидаемых баллов и проверьте:

- правильный, неправильный и пропущенный MCSA;
- все комбинации MCMA с частичным баллом и без него;
- частично и полностью правильный ORDER;
- точное TEXT-совпадение, другой регистр, пробелы, `0` и ручную оценку;
- отрицательный штраф;
- одинаковый итог в UI, TSV/XML и PDF.

Первоисточник: [Scoring Algorithm — TCExam](https://tcexam.org/docs/scoring/).
