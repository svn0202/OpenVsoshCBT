# Безопасность

## Встроенные ограничения

- публичная и административная области разделены по каталогам;
- ресурс требует учётную запись и достаточный уровень доступа;
- тест назначается группам;
- доступ можно ограничить IP-диапазоном и клиентским сертификатом;
- число попыток ограничивается настройкой теста;
- время контролируется сервером;
- действия участника, время реакции и IP сохраняются в журнале;
- вопросы и ответы можно перемешивать для разных участников.

## Усиление в TCExam 17

- уникальный `K_RANDOM_SECURITY` для каждого экземпляра;
- токены PDF не работают с демонстрационным/неизменённым секретом;
- криптографически стойкая генерация кодов;
- удалён legacy-механизм создания переменных из request-параметров;
- ограничены допустимые локальные пути и удалённые хосты;
- исправлены классы SQLi, XSS и IDOR;
- зависимости вынесены в Composer и должны регулярно обновляться.

## Обязательные меры эксплуатации

1. Использовать HTTPS.
2. Сразу удалить стандартного администратора/пароль.
3. Закрыть `install/` после установки.
4. Не публиковать `*/config`, backup, cache и логи.
5. Дать запись только runtime-каталогам.
6. Оставить SELinux `Enforcing`.
7. Ограничить PostgreSQL локальным сервером или точным адресом приложения.
8. Регулярно создавать и проверять резервные копии.
9. Не выводить ошибки и stack trace участнику.
10. Проверять upstream `SECURITY.md` и обновления зависимостей.

## Что не является надёжной защитой

- запрет конкретного браузера или User-Agent;
- блокировка вкладки/правой кнопки;
- скрытый URL без проверки прав;
- доверие только IP при NAT/динамической сети;
- отключение SELinux или TLS ради совместимости;
- секрет, общий для всех установок.

Античит-сигналы могут использоваться для журнала и разбора инцидента, но не
должны автоматически доказывать нарушение без дополнительной проверки.

## Уязвимости

Уязвимость не публикуют в открытом issue. Следуйте процедуре в корневом
`SECURITY.md`. Для собственных изменений OpenVsoshCBT сначала подготовьте
исправление и способ безопасного обновления рабочих площадок.
