# Структура базы данных

TCExam хранит основное состояние в 18 таблицах. Префикс по умолчанию — `tce_`,
но он настраивается при установке.

## Пользователи и доступ

| Таблица | Назначение |
|---|---|
| `tce_users` | учётные записи, уровень, профиль, OTP |
| `tce_user_groups` | группы |
| `tce_usrgroups` | связь пользователей с группами |
| `tce_sessions` | серверные сессии |

## Банк заданий

| Таблица | Назначение |
|---|---|
| `tce_modules` | верхний уровень банка |
| `tce_subjects` | темы внутри модулей |
| `tce_questions` | вопросы, тип, сложность, таймер |
| `tce_answers` | варианты, правильность, позиция |

## Тесты и наборы вопросов

| Таблица | Назначение |
|---|---|
| `tce_tests` | настройки теста, время, баллы, попытки |
| `tce_testgroups` | группы, допущенные к тесту |
| `tce_test_subject_set` | правила выборки вопросов |
| `tce_test_subjects` | темы, входящие в правило выборки |

## Попытки и ответы

| Таблица | Назначение |
|---|---|
| `tce_tests_users` | попытка конкретного пользователя |
| `tce_tests_logs` | вопрос в попытке, текст, балл, время, IP |
| `tce_tests_logs_answers` | выбранные/расположенные варианты |
| `tce_testuser_stat` | агрегированная статистика активности |

`tce_tests_logs` создаётся при формировании попытки и обновляется при ответе.
Уникальная пара «попытка + вопрос» не позволяет создать два независимых лога
одного вопроса в одной попытке.

## Сертификаты

| Таблица | Назначение |
|---|---|
| `tce_sslcerts` | разрешённые клиентские сертификаты |
| `tce_testsslcerts` | связь сертификатов с тестами |

## Источник истины

Для установленной версии структура определяется SQL-файлом конкретной СУБД в
`install/` и миграциями в `install/upgrade/`. Сайт с автогенерированной схемой
может отставать. Перед миграцией снимайте фактическую схему рабочей БД.

Не подключайте TCExam 17 напрямую к legacy-базе TMFCBT: в ней возможны
дополнительные поля и несовместимые типы.

Первоисточник: [Database Documentation — TCExam](https://tcexam.org/docs/database_docs/).
